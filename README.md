# Active Dendritic Field Cell / Активная дендритная клетка

> Experimental GPU benchmark for a neuron-inspired architecture: dendritic state + keyed memory + directional order kernel.
>
> Экспериментальный GPU-бенчмарк архитектуры, вдохновлённой идеей, что один нейрон — это не простая ReLU-точка, а маленькая вычислительная система с внутренними ветками, памятью и специализированными операторами.

---

## TL;DR / Кратко

**English.** ADFC is a small experimental sequence block. It combines three internal mechanisms:

1. **Active dendritic state** — branch-like recurrent state with plateau gates and calcium-like memory.
2. **Keyed memory readout** — a cheap learned key→value retrieval path.
3. **Directional order kernel** — a learned O(T*K) operator for detecting whether event A happened before event B.

**Русский.** ADFC — экспериментальный блок для последовательностей. Он объединяет три механизма:

1. **Активное дендритное состояние** — ветки, gate-плато и calcium-like память.
2. **Keyed memory** — дешёвое адресное чтение key→value.
3. **Directional order kernel** — обучаемый оператор порядка: A-before-B против B-before-A.

Главная идея: вместо `нейрон = activation(Wx)` использовать `нейрон = маленькая внутренняя система`, где разные подмеханизмы отвечают за разные типы вычислений.

## Architecture / Архитектура

### English

ADFC6 processes a token sequence with a compact stack:

```text
tokens
  -> embedding + positional state
  -> active dendritic blocks
  -> keyed memory readout
  -> directional order kernel
  -> classifier
```

The design is intentionally modular:

- **Dendritic state** handles mode, slow context and recurrent accumulation.
- **Keyed readout** handles associative retrieval, such as `query key -> value`.
- **Directional order kernel** handles temporal relations, such as `A happened before B`.

### Русский

ADFC6 обрабатывает последовательность так:

```text
токены
  -> embedding + позиционное состояние
  -> активные дендритные блоки
  -> keyed memory readout
  -> directional order kernel
  -> classifier
```

Модули разделяют роли:

- **Dendritic state** отвечает за режим, медленный контекст и накопление состояния.
- **Keyed readout** отвечает за ассоциативную память: `ключ -> значение`.
- **Directional order kernel** отвечает за порядок событий: `A раньше B`.

## Core mechanisms / Основные механизмы

### 1. Active dendritic state

**English.** Each block keeps branch/rank state `h`. Gates decide how strongly each token writes into internal branch memory. A sigmoid plateau gate and learned decay approximate a calcium-like state.

**Русский.** Каждый блок хранит состояние веток `h`. Gate решает, насколько токен записывается во внутреннюю память. Sigmoid-плато и learned decay имитируют медленное calcium-like состояние.

Pseudo-code:

```python
for t in sequence:
    plateau = sigmoid(gate(x_t, context, h))
    candidate = tanh(value(x_t))
    h = decay * h + (1 - decay) * plateau * candidate
```

### 2. Keyed memory readout

**English.** The model builds keys and values from tokens. The final query reads a value from the most relevant token location. It is not full self-attention over every pair; it is a single query read path.

**Русский.** Модель строит ключи и значения из токенов. Финальный query читает value из подходящей позиции. Это не полный self-attention по всем парам, а один путь адресного чтения.

Pseudo-code:

```python
keys = K(tokens)
values = V(shift(tokens))
query = Q(last_tokens)
read = softmax(query @ keys.T) @ values
```

### 3. Directional order kernel

**English.** The weak point of ADFC3 was event order. ADFC6 adds learned detectors for event-like signals and computes a directional relation:

```text
order = A_before_B - B_before_A
```

This is O(T*K), not O(T^2). It uses prefix sums.

**Русский.** Слабым местом ADFC3 был порядок событий. ADFC6 добавляет обучаемые детекторы событий и считает направленное отношение:

```text
order = A_before_B - B_before_A
```

Это O(T*K), а не O(T²), потому что используется prefix-sum.

Pseudo-code:

```python
a_t = detector_A(x_t)
b_t = detector_B(x_t)
A_before_t = prefix_sum(a) before t
B_before_t = prefix_sum(b) before t
order = sum_t b_t * A_before_t - sum_t a_t * B_before_t
```

## Synthetic tasks / Синтетические задачи

### `mode_select`

**English.** A mode token decides whether the answer should come from the A-marked value or the B-marked value.

**Русский.** Режим в начале последовательности говорит, брать значение после A-маркера или после B-маркера.

Tests: conditional routing + memory.

### `kv_recall4`

**English.** Four key-value records are placed in the sequence. The final query asks for one key. The model must return the associated value.

**Русский.** В последовательности лежат 4 записи key-value. В конце query спрашивает один ключ. Модель должна вернуть связанное значение.

Tests: associative retrieval.

### `order_compare`

**English.** Events A and B appear in random order. The model must decide whether A happened before B, with an optional mode flip.

**Русский.** События A и B появляются в случайном порядке. Модель должна определить, было ли A раньше B, с возможным инвертирующим mode.

Tests: temporal order.

## Results / Результаты

### Confirmed previous runs / Подтверждённые предыдущие прогоны

| Task | Mean pool | ADFC3 | Margin |
|---|---:|---:|---:|
| mode_select | 75.78% | 100.00% | +24.22% |
| order_compare | 51.66% | 54.20% | +2.54% |
| kv_recall4 | 71.48% | 100.00% | +28.52% |

ADFC6 order run, larger config:

| Task | Mean pool | ADFC3 | ADFC6 |
|---|---:|---:|---:|
| order_compare | 51.37% | 86.96% | **99.85%** |

Interpretation:

- ADFC3 solved mode selection and key-value recall.
- ADFC6 solved the remaining temporal-order weakness.

Интерпретация:

- ADFC3 закрыл режимы и key-value recall.
- ADFC6 закрыл слабое место с порядком событий.

## Why this is interesting / Почему это интересно

**English.** Standard neural networks often treat a neuron as a scalar nonlinearity. Biology suggests a cell may contain rich internal computation: compartments, active dendrites, slow state, gates, and multiple signaling modes. ADFC tests a minimal machine-learning version of that idea.

**Русский.** Обычные нейросети часто считают нейрон простой scalar activation. Биология намекает, что клетка может быть намного сложнее: compartments, активные дендриты, медленное состояние, gate-механизмы, разные каналы передачи. ADFC проверяет минимальную ML-версию этой идеи.

Important: this is **not** a claim that the model reproduces biology. It is an engineering abstraction inspired by biological mechanisms.

Важно: это **не** утверждение, что модель точно копирует биологию. Это инженерная абстракция, вдохновлённая биологическими механизмами.

## What is unique and what is not / Что уникально, а что нет

Not unique:

- recurrent state
- attention-like readout
- event detectors
- synthetic sequence tasks

Potentially interesting combination:

- dendritic branch state
- keyed single-query memory
- directional prefix-order kernel
- unified benchmark showing which mechanism solves which task

Не уникально:

- recurrent state
- attention-like readout
- event detectors
- synthetic sequence tasks

Интересная комбинация:

- дендритное состояние веток
- single-query keyed memory
- directional prefix-order kernel
- единый бенчмарк, показывающий, какой механизм какую задачу закрывает
